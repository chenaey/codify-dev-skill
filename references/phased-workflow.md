# 复杂设计流程

## 核心约束

禁止一次性生成所有代码。

---

## 流程

```
理解 → 规划 → 逐个实现 → 组装
```

### 1. 理解设计

获取数据后，先建立整体认知：

- 设计包含哪些主要区域？
- 各区域的功能和层级关系？
- 哪些结构可以抽象为组件？

> 💡 根节点的直接子节点通常对应独立的功能区域

### 2. 规划组件

基于理解，创建 `structure.md`：

```markdown
| 组件 | 节点 ID | 状态 |
|------|---------|------|
| Header | 377:68199 | ⬜ |
| ProductCard | 500:028698 | ⬜ |
```

状态：⬜ 待实现 → ✅ 已完成

### 3. 逐个实现

循环直到所有组件 ✅：
1. 生成代码（遵循 [codegen-rules.md](codegen-rules.md)）
2. 下载资源
3. 更新状态

### 4. 组装

创建页面入口文件，组合所有组件。
